AI-Powered Document Assistant (RAG Application)

Enterprise-ready chatbot developed for Eagles Wing Limited, enabling instant Q&A over internal documents using Retrieval-Augmented Generation.

Objective
Automate document search and knowledge retrieval for internal teams
Timeline
2-Week Rapid Development Sprint
My Role
Full-Stack AI Developer
Tech Stack
Python, LangChain, OpenAI API, Streamlit, ChromaDB, PyPDF
What It Does
  • Allows staff to upload PDF documents and ask natural-language questions about their content.
  • Uses Retrieval-Augmented Generation (RAG) to locate relevant sections and produce concise, cited answers.
  • Processes PDFs into searchable chunks with OpenAI embeddings stored in ChromaDB for fast retrieval.
  • Provides a secure, browser-based interface built in Streamlit with real-time response feedback.
Technical Implementation
  • Implemented a full ingestion pipeline using LangChain & PyPDFLoader for parsing and semantic chunking (1 000-token size, 200 overlap).
  • Optimised retrieval accuracy by fine-tuning k = 3 top results and a zero-temperature LLM for consistent factual output.
  • Handled compatibility across LangChain sub-modules (langchain_community, langchain_text_splitters), ensuring stable production builds.
  • Deployed live prototype with secure API-key management and clear user documentation for internal adoption.
Business Impact & Use Cases
  • Eliminated manual document search, saving the client’s team hours of administrative time weekly.
  • Enabled quick answers for legal, HR and compliance documents through conversational AI queries.
  • Demonstrated practical AI integration and deployment capabilities aligned with UK tech-sector hiring trends.
Key Achievements
  • Delivered a functional RAG system within a 2-week sprint, fully integrated into the client’s workflow.
